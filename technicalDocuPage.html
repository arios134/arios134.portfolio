HTML Tables

Formatting a table requires that each row and column be defined, in addition to defining what content will be displayed in each cell.

Much like pictures, CSS properties can be used to style the table so that it can have borders, different colors, padding and other properties.

Tables are defined with the <table> tag, each table row is defined with the <tr> tag and each table header in that row is defined with the <th> tag. To add data into each cell, use the <td> tag. Nesting is important to format the table. The table below will have 10px padding between cell content and its borders, a border added and collapse those borders into one. You can see what the table looks like without border collapse by adding a comment.

    Key Points to Remember
  • For style properties to apply to all tables in the page, specify properties in the head section
  • Make sure to specify which tags will get the defined property
  • Style changes applied within the table will override the default property in the header

Note, extra spaces added in tags to show the HTML format with rendering below
<table id="t01" style="width:100%"> <tr> <th>Firstname</th> <th>Lastname</th> <th>Age</th> </tr> <tr> <td>James</td> <td>Smith</td> <td>35</td> </tr> <tr> <td>Martha</td> <td>Zales</td> <td>29</td> </tr> </table>
Firstname Lastname Age
James Smith 35
Martha Zales 29
Unordered Lists

In an unordered list, each list item is designated with a bullet point by default. This marker can be changed with CSS list-style-type property and be defined as a bullet (default), disk, square, circle or none.

To start an unordered list, use the <ul> tag with each <li> nested within. Items with descriptions can also be added for more complex lists.

    Key Points to Remember
  • Items added to the list will be in no particular order
<ul style="list-style-type:square;"> <li>Coffee</li> <li>Tea</li> <li>Milk</li> </ul>
  • Coffee
  • Tea
  • Milk

Ordered Lists

An ordered list will automatically number each list item. The numbering sequence can be changed between upper/lowercase letters or roman numerals

    Key Points to Remember
  • Can change the starting count sequence by changing the <start> attribute
  • Detailed lists that name on item and give a description are also supported
  • Nesting your code properly matters!
<ol type="I"> <li>Coffee</li> <li>Tea</li> <li>Milk</li> </ol>
  1. Coffee
  2. Tea
  3. Milk

Input Forms

A form can be created using the form element so that users can input information and submit. What happens to the input data is up to you through the action attribute.

In order to create a field that will take in data, you will need to include the <input> element and set the type attribute to the data type expected to be input.

    Key Points to Remember
  • Many data type values that can be used. Some examples are text, datetime-local, password, radio and number
  • A placeholder attribute can also be added to describe the input field, its value takes a string
  • A required attribute means that the user must enter data in the field in order to submit the form.
<form action=""> <input type="text" required placeholder="Type your name"> <button type="submit">Submit</button> </form>

Form Datasets

Datasets allow grouping of related information presented in a form. This provides greater readability for those using assistive technology to access your site. This is accomplished by using two tags nested in the form element. The fieldset tags groups related elements in a form and draws a box around the items. The legend tag defines a caption for the fieldset element.

This feature is also useful in keeping related form data together when users submit their information online. The data can then be placed into a database for future use. <form action="" method="POST"> <fieldset> <legend><span>Select which food item you prefer</span></legend> <label for="food"> <input id="food" type="radio" name="favfood">Pizza </label> <label for="food"> <input id="food" type="radio" name="favfood">Sushi </label> </fieldset> </form>
